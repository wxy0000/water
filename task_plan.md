# Hydropace 代码瘦身计划

## 目标
从第一性原理审计 Hydropace 当前需求、文档和代码，删除或合并不必要的代码与逻辑，同时保持喝水记录、桌面浮窗、菜单栏、设置、系统通知这些核心功能可运行。

## 边界
- 优先删除明显冗余、重复、过度诊断、过早抽象和失效文档。
- 不改变用户可见核心流程，除非能减少复杂度并通过测试验证。
- 不回滚用户或前序工作中的有效修改。
- 对 macOS 通知相关代码保持谨慎，因为它刚刚完成真实系统级排障。

## 阶段
1. [complete] 盘点文档与源码结构，建立功能到代码映射。
2. [complete] 找出冗余候选：未使用文件、重复逻辑、过度注释、旧品牌残留、过重 UI/状态逻辑。
3. [complete] 提出精简设计与取舍，确认可删范围。
4. [complete] 按测试保护实施小步删除/合并。
5. [complete] 运行验证：Rust 测试、TypeScript 构建、前端/e2e、Tauri 构建。
6. [complete] 记录结果和剩余风险。

## 当前假设
- 这个 App 的核心价值是低摩擦记录喝水和可靠提醒；不应为尚未存在的复杂功能保留大量代码。
- 文档可能有阶段式开发遗留，代码也可能有调试阶段遗留。

## 结果
- 删除低风险冗余：未使用前端依赖、未使用 Cargo 直接依赖、旧 widget commands、旧 widget 前端 DAO/drag hook、过期阶段注释。
- 保留通知原生桥、趋势图、历史列表等用户可见功能；这些若要继续瘦身，需要产品层取舍。
- 验证通过：`npm run build`、`cargo test`、`npm run test:e2e`、`npm run tauri:build`。
- 未运行：`cargo clippy` / `cargo fmt`，因为当前 stable toolchain 未安装对应组件。
