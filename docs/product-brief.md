# L01 Water Reminder — 产品简介

## 一句话定位

桌面级轻提醒喝水助手——简约、美观、丝滑 spring 动画 + 智能提醒。

## 目标用户

**通用忙碌白领**（不限定本人，是典型"类型 1-3"用户）：

- 长时间坐在电脑前工作
- 工作时注意力被占，容易忘记喝水
- 对弹窗 / 通知敏感（不想被频繁打扰）
- 追求工具的视觉品质（看不上粗糙 UI）
- 已经在用 Mac / Windows 桌面办公

## 核心痛点

1. **工作时忘了喝水** —— 注意力资源被完全占用，喝水变成背景需求
2. **现有 App 通知太频繁** —— 反而被通知淹没，关键时刻漏掉
3. **现有 App UI 粗糙 / 过度装饰** —— 不愿每天打开，3 天即卸
4. **需要打开浏览器 / 手机 App** —— 摩擦太大，记完就忘

## 解决方案

**桌面菜单栏小工具**（macOS 优先，Windows 次优）：

- 菜单栏常驻图标，显示今日进度
- 点击弹出极简 popover（不用打开浏览器 / App）
- 进度圆环 + 一键记录（3 个杯子按钮：小 / 中 / 大）
- 自动智能提醒：每 60-90 分钟，仅工作时间
- 动画：spring physics / 非线性 easing
- 数据本地存储，无账号、无后端、无云同步

## 用户洞察（来自本人自我访谈）

> 注：以下洞察来自本项目唯一用户（本人）的 4 块自我访谈。
> 本 MVP 不针对本人（本人属于"对所有主动操作都抗拒"型，App 几乎无效），
> 但访谈过程中的方法论发现值得记录。

**关键发现**：

- 通用用户喝水少 = 第 1-5 类（公开数据已覆盖）
- 本人喝水少 = 第 1+4+6+7 类（复合型，第 6/7 类为访谈新增）
- 第 6 类：主动抗拒提醒型（提醒会引发反向反应）
- 第 7 类：抗拒所有主动操作型（连大水壶都嫌清理麻烦）

**对 MVP 的影响**：

- 即使本人用不上，这个 MVP 对通用用户（第 1-5 类）有价值
- 课程作业交付的是"AI 做产品流程" + "MVP 跑通"，不是"自用价值"
- 7 天验证如果本人用不上，**这是诚实的第一性结论，不是失败**

## 不做什么（明确排除）

- ❌ 频繁通知（每 30 分钟弹一次）—— 通知疲劳
- ❌ 强制弹窗（必须操作才能关）—— 反向激怒
- ❌ 进度条 / 虚拟植物装饰 —— 新鲜感衰减
- ❌ 排行榜 / 社交 / 挑战 —— 外部激励衰减
- ❌ 喝水声效 / 游戏化 —— 噪音
- ❌ 集成 Apple Health / Google Fit —— 复杂度无收益
- ❌ 账号系统 / 云同步 —— 增加摩擦

## 成功标准

| 维度 | 标准 |
|---|---|
| 功能 | 7 天内平均日饮水量 ≥ 1500ml（本人预期做不到，验证第 7 类） |
| 体验 | 7 天内用户没有卸载冲动（< 3/10） |
| 视觉 | UI 被评价为"美观"（自评 + 戴师兄反馈） |
| 流畅 | spring 动画无卡顿、不突兀 |
| 协作 | AI 协作日志完整记录每段代码生成过程 |
