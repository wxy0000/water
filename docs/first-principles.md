# 第一性原理：为什么人不主动喝水

## 表面问题

> 人忘记喝水 → 提醒 → 解决

## 真实问题（5 类，基于公开数据）

| 类型 | 占比 | 现有 App 解 | 失效原因 |
|---|---|---|---|
| 1. 注意力被占，忘了 | 40% | 定时通知 | 通知疲劳 |
| 2. 不知道喝多少合适 | 25% | 设定目标量 | 粗糙锚点（"8 杯"） |
| 3. 没反馈，不知道够不够 | 20% | 进度条 / 虚拟植物 | 短期新鲜，长期无感 |
| 4. 动机弱，不在乎 | 10% | 排行榜 / 挑战 | 外部激励衰减 |
| 5. 物理不便（手边没水） | 5% | 无 | 需要环境设计 |

## 关键判断

1. **通知疲劳** 让"提醒"对 40% 用户失效。
2. **真正未充分解决**：类型 1-3（约 85%）。
3. **MVP 应避开"通知疲劳"陷阱**：轻量、不频繁、不强提醒。

## MVP 方向

**桌面级轻提醒小助手**：

- 智能提醒（每 60-90 分钟，仅工作时间）
- 极简记录（一键，不输数字）
- 美观 UI（极简 + spring 动画）
- 不做：进度条 / 虚拟植物 / 排行榜 / 社交 / 强制提醒

## 个人访谈扩展（2026-06-28，本项目唯一用户）

对本人进行的 4 块自我访谈发现：本人属于**复合类型**（1+4+6+7），对所有"主动操作"型方案抗拒。但作为课程作业，本 MVP 针对**通用用户**（类型 1-5 典型），不针对本人。
