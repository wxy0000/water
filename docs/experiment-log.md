# 实验日志：7 天自我验证

## 实验设计

- **周期**：7 天
- **主体**：自己（唯一用户）
- **核心问题**：我的喝水行为是否真的改变？
- **次要问题**：哪个功能被真用 / 被绕过 / 有副作用？

## 关键指标

### 行为指标（自评 + 自报）

| 指标 | 测量方式 | 频率 |
|---|---|---|
| 日均喝水量（ml） | 自测估计，每晚记 | 每日 |
| 主动想起喝水的次数 | 自报 | 每日 |
| 实际记录次数 | App 后台日志 | 自动 |
| 卸载冲动（0-10） | 每晚打分 | 每日 |
| 主动想用 App 的次数 | 自报 | 每日 |

### 体验指标

- 哪个按钮被反复用？（一杯 / 半杯 / 没喝）
- 是否有"想看趋势但还没解锁"的焦虑？
- 是否有"今天漏记"的挫败感？

## 复盘维度（7 天后填）

1. **行为是否真改变？**
   - 不仅是数字，是"我喝水时的思考方式"
2. **哪个功能被真用？哪个被绕过？**
   - 例：如果"半杯"按钮几乎不用，说明它不该存在
3. **哪个功能有副作用？**
   - 例：如果"延迟解锁趋势"让人焦虑，应该改为立即可看
4. **哪些功能应该加？哪些应该删？**
5. **如果再做一次，MVP 会改成什么样？**
